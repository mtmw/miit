Doorbell 门铃
主板上串口顺序: 3V3、GND、TX、RX 波特率 115200
对应USB串口线顺序: 红、黑、白RX、绿TX
注意: 电源可不接 地线一定要接

USB座
VBUS Red（红色）
D- White（白色）
D+ Green（绿色）
GND Black（黑色）

RXD Receive Data 为接收数据的引脚
TXD Transmit Data 为发送数据的引脚

门铃上电后启动时出现 Hit any key to stop autoboot:
立即按任意键进入u-boot模式: 
printenv                           # 显示当前环境变量 如果不匹配请做修改
setenv ipaddr 192.168.3.130        # 设置门铃设备的IP 依开发计算机的网段来设置
setenv serverip 192.168.3.95        # 设置tftp服务器(即开发计算机)IP 网段为192.168.3.xxx    ipconfig /all 内网IP查询
ping 192.168.3.95                   # ping开发计算机的IP 以测试门铃设备的网络有没有alive
# host 192.168.3.95 is alive
启动 tftpd32.exe,单击 Browse 选择 img 文件的路径 
mw.b 0x82000000 0xff 0x1000000                                     # 恢复16M
tftp 0x82000000 Hi351x-16M-ipc4206p-bell.img                       # FTP下载img固件到DDR  .sw是web升级文件
sf probe 0;sf erase 0 0x1000000;sf write 0x82000000 0x0 0x1000000;   # 擦写FLASH
re                                                                 # reset

# 如果有多条命令要写到一行，中间可以用冒号(;)隔开，冒号前后不必空格

外接网线颜色1-6依次为:
蓝白(LAN_TX+)、蓝(LAN_TX-)、绿白(LAN_RX+)、绿(LAN_RX-)、褐(LED1_LINK)、紫(LED1_ACT)

IPCSearch 状态:
状态栏中是 + 加号：表示是新增加的，上一次搜索没搜索到的，这次搜到了
状态栏中是 - 减号：表示离线
状态栏中是空的，表示连接正常

无线升级IP 192.168.234.1
