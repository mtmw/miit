域名 Domain Name 
A 记录        域名对应的 IPV4 IP 地址
AAAA 记录     域名的 IPV6 IP 地址,目前很多域名并没有 IPV6 地址
NS 记录       域名的名称服务器 Name Server 
CNAME 记录    也叫别名记录.这个记录是把一个域名指向另一个域名
              例如给 abc.com设置一个 bbs.abc.com 的 CNAME 记录，把这个记录设置为 www.abc.com，那么访问
              bbs.abc.com 实际上就是要去访问 www.abc.com。注意这个和单纯的跳转不同，应该理解为转译

DNS解析全过程 
dig +trace baidu.com        # 执行后分成了4个部分，
                            # 第一部分是根，也就是.(点) DNS会返回“根域名服务器”地址和IP
                            # 第二部分是我们com后缀的权威DNS  返回gTLD地址  根域名服务器返回通用顶级域名（gTLD）的服务器地址和IP
                            # 第三部分是从com到baidu域名的权威DNS  gTLD服务器返回NS（name server）地址和IP
                            # 第四部分是从百度的权威DNS得到正常查询结果  NS返回域名对应的真正的IP地址

Tracert ip        # 可以看到各个节点的延迟和丢包 Windows版本是winmtr
